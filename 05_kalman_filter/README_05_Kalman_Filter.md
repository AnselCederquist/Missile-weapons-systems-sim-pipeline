# Project 05 - 6-State Extended Kalman Filter

State estimation for a boost-sustain missile trajectory. EKF fuses simulated IMU accelerometer measurements and GPS position/velocity to estimate full 6-state trajectory. 
Ground truth trajectory integrates equations of motion with drag from the Project 04 DATCOM aerodynamic database.

## Overview

Real missile guidance systems cannot directly observe their own state. Onboard sensors are noisy: IMUs drift and accumulate error, GPS has position uncertainty, and neither gives a clean picture of where the missile is 
or how fast it is moving at any instant. A Kalman filter solves this by fusing multiple noisy measurements with a dynamic model of the system to produce a minimum-variance state estimate -- the best possible estimate 
of position and velocity given all available information.

This project implements a 6-state Kalman filter for a boost-sustain tactical missile trajectory. The filter fuses simulated IMU accelerometer measurements (100 Hz, sigma = 0.5 m/s^2) with simulated GPS position and 
velocity (10 Hz, sigma = 5 m / 0.1 m/s) to estimate the full position and velocity state vector in 3D. Ground truth is generated by numerically integrating the equations of motion under a two-phase boost-sustain 
thrust profile, with aerodynamic drag computed at each timestep from the Project 04 DATCOM aerodynamic database via `aero_interpolator.get_CD(mach, alpha)`. This makes Project 05 a direct downstream 
consumer of Project 04 -- the pipeline is live.

The Kalman filter operates in two steps at each timestep. In the predict step, the current state estimate is propagated forward using the kinematic equations of motion with IMU acceleration as the input -- 
position updates from velocity, velocity updates from measured acceleration minus gravity. Uncertainty grows during prediction because IMU measurements are noisy. In the update step (at GPS rate, 10 Hz), 
the GPS measurement is compared against the predicted state, and the Kalman gain matrix determines how much to trust the measurement versus the prediction. A well-tuned filter weights GPS heavily when 
prediction uncertainty is high and lightly when the model is trusted -- the residuals (EKF minus truth) should be zero-mean and bounded.

Results: Position RMSE 0.98 m against a ~10.5 km range trajectory. Velocity RMSE 0.39 m/s. Residuals zero-mean, bounded within +/-3 m, no secular drift -- filter is consistent and well-tuned. The state 
estimate feeds directly into the Project 06 6-DOF flight mechanics simulator as the navigation solution.

## Why This Matters for GNC

Navigation filters are the core of any guided munition. Without accurate state estimation, guidance laws cannot compute correct control commands and the missile misses. The Kalman filter is the 
industry-standard solution -- variants of this algorithm run in every modern precision-guided weapon, satellite, and autonomous vehicle. Implementing one from scratch, validating it against a 
realistic trajectory with real sensor noise models, and integrating it with an aerodynamic database demonstrates the full GNC stack from aero to navigation to simulation.

## EKF Design

- Process model: constant acceleration with IMU as input
- State transition matrix F: kinematic 6x6 with dt coupling
- Process noise Q: diagonal, tuned to IMU noise characteristics
- Measurement matrix H: identity (direct state observation via GPS)
- Drag computed from Project 04 aero_interpolator.py at current Mach number

## Design Considerations and Limitations

**Linear vs Extended KF:** This implementation uses a linear Kalman filter with kinematic state transition. True EKF would linearize the nonlinear drag force equations via Jacobians at each timestep. The drag nonlinearity is handled here by feeding IMU-measured acceleration directly as the process input rather than modeling drag analytically in the state transition -- this is a common simplification for navigation filters where IMU is the primary propagation source.

**IMU as process input:** Accelerometer measurements drive the predict step rather than a dynamic model. This means filter performance degrades if IMU noise is increased significantly -- the process model has no knowledge of thrust or drag, only measured acceleration. Appropriate for a navigation filter; a guidance filter would require full dynamics.

**GPS update rate:** 10 Hz GPS updates on a 100 Hz propagation loop. Residuals show slightly higher variance in the first 2 seconds during the high-acceleration boost phase -- GPS latency relative to rapid velocity change. Increasing GPS update rate to 100 Hz during boost would reduce initial transient.

**Y-axis:** Trajectory is planar (XZ). Y-state and Vy are propagated but trivially zero -- filter correctly estimates near-zero with noise bounded by GPS sigma. Excluded from plots for clarity.

**Drag model:** CD pulled from DATCOM database at alpha=0 and current Mach. At high AoA during boost the drag is underestimated -- acceptable for a navigation study but would require full aero_interpolator.get_CD(mach, alpha) call with attitude estimate for higher fidelity.

**No attitude states:** 6-state position/velocity filter only. Full GNC implementation would add 3 attitude states (roll, pitch, yaw) and 3 gyro bias states for a 9 or 15-state filter. Attitude estimation left for Project 06 integration.
